# Publish codex_project Backend To AWS

This folder contains an AWS SAM backend for the `codex_project` data generated by
the Colab pipeline.

## What Gets Deployed

- API Gateway: `codex_project-api`
- Lambda functions:
  - `codex_project-similarity-search`
  - `codex_project-get-experiences`
  - `codex_project-library-api`
  - `codex_project-admin-api`
  - `codex_project-dashboard-initiator`
- DynamoDB tables matching the Colab pipeline defaults:
  - `codex_project-content`
  - `codex_project-books`
- Static UI S3 bucket:
  - `codex-project-ui-<account-id>-<region>`
- CloudFront distribution for HTTPS UI hosting
- A CloudFormation custom resource that writes `config.json` into the UI bucket
- React motivation search UI from `../frontend`

## Prerequisites

Install and configure:

```bash
brew install awscli aws-sam-cli python@3.12
aws configure
```

Or use any equivalent AWS CLI/SAM installation.

Your AWS identity needs permission to create:

- CloudFormation stacks
- Lambda functions and IAM roles
- API Gateway REST APIs
- DynamoDB tables
- S3 buckets and bucket policies
- CloudFront distributions

## Required Environment Variables

The Makefile automatically loads defaults from the root `../.env` file when it
exists. You can either edit that file or export variables in your shell.

For search to work, set Qdrant and Hugging Face credentials before deploy:

```bash
export QDRANT_URL="https://your-cluster-url"
export QDRANT_API_KEY="your-qdrant-api-key"
export HF_API_KEY="your-huggingface-token"
```

The Lambda uses Hugging Face Inference API with `BAAI/bge-m3` to embed user
queries so they match the BGE-M3 vectors created in Colab.

Optional overrides:

```bash
export AWS_REGION="ap-south-1"
export STACK_NAME="codex-project-backend"
export VIDEO_DYNAMO_TABLE="codex_project-content"
export BOOK_DYNAMO_TABLE="codex_project-books"
export VIDEO_QDRANT_COLLECTION="codex_project-videos"
export HF_EMBEDDING_MODEL="BAAI/bge-m3"
```

If the DynamoDB tables already exist because you created them manually, deploy
with:

```bash
export CREATE_DYNAMO_TABLES=false
```

If this stack should create the tables, keep the default:

```bash
export CREATE_DYNAMO_TABLES=true
```

## Important Data Alignment

The Colab pipeline defaults are:

```text
Video DynamoDB table: codex_project-content
Book DynamoDB table:  codex_project-books
Video Qdrant collection: codex_project-videos
Book Qdrant collection:  codex_project-books
```

This backend reads the same video/book DynamoDB table names. The `/search`
endpoint searches `codex_project-videos`.

For hybrid BM25 search, copy the Colab-generated BM25 artifact into:

```text
cloud-backend/lambdas/similarity_search/vocab_idf.json
```

If you do not copy it, `/search` still works in dense-only mode.

## Deploy With Make

From the repo root:

```bash
cd cloud-backend
make validate
make build
make deploy
```

`make deploy` is non-interactive. It loads defaults from `../.env` and the
Makefile, then passes them to SAM with `--parameter-overrides`.

If you explicitly want SAM's guided prompts:

```bash
make deploy-guided
```

For repeat deployments plus UI upload:

```bash
make release
```

For backend-only refresh plus UI upload:

```bash
make refresh
```

To upload only the frontend/static UI:

```bash
make upload_ui
```

If `../frontend/package.json` exists, the Makefile runs `npm ci` when a lockfile
exists, otherwise `npm install`, then `npm run build`, and uploads
`../frontend/dist`. If no frontend exists, it uploads the fallback UI in
`cloud-backend/ui/`.

`upload_ui` also writes a fresh `config.json` into the UI bucket:

```json
{
  "projectName": "codex_project",
  "apiUrl": "https://your-api-id.execute-api.region.amazonaws.com/prod"
}
```

The React UI reads this file at runtime. For local frontend development, set:

```bash
cd ../frontend
VITE_API_URL="https://your-api-id.execute-api.us-east-1.amazonaws.com/prod" npm run dev
```

To print stack outputs:

```bash
make outputs
```

To delete the stack:

```bash
make teardown
```

## API Endpoints

After deploy, `make outputs` prints `ApiUrl`.

Routes:

```text
GET /search?q=<query>
GET /experiences
GET /videos
GET /videos/{videoId}
GET /books
GET /admin/videos
GET /admin/videos/{videoId}
PUT /admin/videos/{videoId}
```

All Lambda responses include CORS headers for browser clients.
